# Landing - Handoff Checklist

## Definition Of Done (landing)

- [ ] Header coherent desktop/mobile.
- [ ] Hero aligne avec promesse produit et CTA lisibles.
- [ ] Sections features homogenes (grille, espacements, tags).
- [ ] Bloc integrations stable (cards/logo states).
- [ ] Bloc temoignage stable (controls etats).
- [ ] CTA final coherent avec hero CTA.
- [ ] Footer harmonise (typo/espacement/liens).

## Components And Variants

- [ ] Reuse gate applique avant tout nouveau composant.
- [ ] `component-registry.md` mis a jour.
- [ ] Etats CTA couverts: `default`, `hover`, `focus`, `disabled`, `loading`.
- [ ] Etats controles couverts (carousel/navigation).

## Responsive And A11y

- [ ] Variantes desktop + mobile pour sections critiques.
- [ ] Contraste textes secondaires valide.
- [ ] Focus visible sur elements interactifs.
- [ ] Cibles tactiles mobiles coherentes.

## Handoff Required

- [ ] Liste composants reutilises.
- [ ] Liste variantes creees.
- [ ] Liste nouveaux composants + raison.
- [ ] Mapping section -> module (landing: `M13` par defaut).
- [ ] Contraintes backend/no-code rappelees.
