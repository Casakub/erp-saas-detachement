# Design Tokens - Spacing

## Rule

- Utiliser une echelle de spacing unique pour tout le systeme.

## Token Families

- `space.0|1|2|3|4|6|8|10|12|16|20|24`
- `radius.none|sm|md|lg|xl|full`
- `shadow.sm|md|lg`

## Usage

- Eviter les valeurs libres hors echelle.
- Utiliser la meme logique desktop/mobile avec adaptations explicites.
